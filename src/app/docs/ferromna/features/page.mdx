# Features

FerroMNA provides a comprehensive simulation capability covering analog, digital, RF, and mixed-signal circuit design. This page is the complete reference for all supported analysis types, device models, and convergence algorithms.

## Analysis Types (16)

| Analysis | SPICE Card | Description |
|---|---|---|
| DC Operating Point | `.op` | Newton-Raphson iteration with convergence aids: GMIN stepping, source stepping, line search |
| DC Sweep | `.dc` | Sweeps a source across a range; single or dual source support |
| AC Small-Signal | `.ac` | Frequency sweep (DEC/LIN/OCT) with complex MNA formulation |
| Transient | `.tran` | Time-domain with Backward Euler/Trapezoidal integration and adaptive LTE timestep control |
| Harmonic Balance | `.hb` | Multi-tone RF analysis in the frequency domain via FFT |
| Noise | `.noise` | Output noise spectral density using the adjoint method |
| Pole-Zero | `.pz` | Complex-plane pole/zero extraction via eigenvalue decomposition |
| Sensitivity | `.SENS` | DC and AC parameter sensitivity (direct method, ngspice-compatible) |
| Optimization | `.OPTIMIZE` | Gradient-descent circuit optimization with weighted performance targets |
| Monte Carlo | `.MC` | Statistical yield analysis with Gaussian and uniform parameter tolerances |
| FFT / Spectral | — | Frequency spectrum with Hann, Hamming, and Blackman windowing options |
| S-Parameter | `.SP` | S-parameter extraction with Z/Y/S conversions and mixed-mode port support |
| X-Parameters | — | Large-signal characterization via harmonic balance perturbation |
| Distortion | `.DISTO` | HD2/HD3 and IM2/IM3 harmonic and intermodulation distortion |
| Transfer Function | `.TF` | DC gain, input impedance, and output impedance |
| Step Sweep | `.STEP` | Parameter step sweep (LIST or START/STOP/INC) |

## Convergence Algorithm

FerroMNA uses a six-stage convergence strategy with automatic fallback. Each stage is tried in order; if a stage fails to converge, the next stage is invoked automatically.

1. **Direct Newton-Raphson** — Chord method with LU factor reuse; the fastest path for well-conditioned circuits
2. **GMIN Stepping** — Alan Gillespie algorithm: injects a conductance from 1×10⁻² → 1×10⁻¹² S and solves at each step
3. **Source Stepping** — Adaptive source ramp from zero with state save/restore at each step
4. **OPTran Fallback** — Raised-cosine source ramp with automatic timestep halving
5. **Line Search** — Applies Armijo damping when the maximum voltage update exceeds 10 V in a single NR step
6. **Limit Cycle Detection** — Detects oscillating NR iterations using a 3-iteration history and averages the state to break the cycle

## Performance Benchmarks

Measured on a 12-core Intel Core i7 machine, single-threaded:

| Circuit | FerroMNA | ngspice | Speedup |
|---|---|---|---|
| RCA3040 (11 BJTs, 50 nodes) — transient | 89 ms | 293 ms | **3.3×** |
| RC charging — transient | 0.78 ms | 1.0 ms | **1.3×** |
| BSIM4 common-source — AC | 0.61 ms | 1.0 ms | **1.6×** |
| MOSFET Level 3 common-source — AC | 0.33 ms | 1.0 ms | **3.1×** |

Performance varies by circuit topology, model complexity, and analysis type. BJT-heavy circuits with dense node networks show the largest speedups due to FerroMNA's efficient Jacobian evaluation and sparse solver.
