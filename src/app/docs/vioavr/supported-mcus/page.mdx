# Supported MCUs

VioAVR supports over 340 MCU variants across the full range of AVR microcontroller families. Each variant has a complete device descriptor generated from official Microchip ATDF device pack files, ensuring accurate memory sizes, register layouts, peripheral configurations, and interrupt vector tables. The registry below highlights key supported variants from each series.

## Adding New MCU Variants

To add support for a new MCU variant:

1. Obtain the ATDF file from Microchip's Device Family Pack or the Atmel Packs repository.
2. Run `gen_device_descriptor.py` to generate the C++ device descriptor from the ATDF.
3. Add the generated descriptor to the device registry in the VioAVR source tree.
4. Update the CMake build system to include the new descriptor in compilation.
5. Test the new variant with `dump_device_descriptor` and run a simple blink firmware to verify basic operation.

```bash
# Example: Adding ATmega328PB support
gen_device_descriptor.py ATmega328PB.atdf --output-dir src/devices/atmega328pb

# Verify the generated descriptor
dump_device_descriptor atmega328pb --all

# Test with a simple firmware
vioavr-cli --mcu atmega328pb --flash blink.hex --max-cycles 1000000
```
